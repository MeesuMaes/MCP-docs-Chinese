本指南将帮助您使用大语言模型（LLM）来构建自定义的模型上下文协议（MCP）服务器和客户端。在本教程中，我们将以Claude为例，但您可以使用任何前沿的大语言模型。

## 准备文档

在开始之前，收集必要的文档以帮助Claude理解MCP：

1. 访问 https://modelcontextprotocol.io/llms-full.txt 并复制完整的文档文本。
2. 导航到[MCP TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) 或 [Python SDK 仓库](https://github.com/modelcontextprotocol/python-sdk)。
3. 复制README文件及其他相关文档。
4. 将这些文档粘贴到与Claude的对话中。

## 描述您的服务器

提供文档后，清楚地向Claude描述您希望构建的服务器类型。具体说明以下内容：

- 您的服务器将暴露哪些资源。
- 它将提供哪些工具。
- 它应包含哪些提示。
- 需要与哪些外部系统交互。

例如：
```
构建一个MCP服务器，满足以下需求：
- 连接到我公司的PostgreSQL数据库。
- 将表结构作为资源暴露。
- 提供运行只读SQL查询的工具。
- 包含用于常见数据分析任务的提示。
```

## 与Claude协作

在与Claude协作构建MCP服务器时：

1. 从核心功能开始，然后逐步添加更多功能。
2. 如果您不理解代码的某些部分，请让Claude为您解释。
3. 根据需要请求修改或改进。
4. 让Claude帮助您测试服务器并处理边缘情况。

Claude可以帮助实现所有关键的MCP功能：

- 资源管理与暴露。
- 工具定义与实现。
- 提示模板与处理器。
- 错误处理与日志记录。
- 连接与传输设置。

## 最佳实践

在使用Claude构建MCP服务器时：

- 将复杂的服务器分解为较小的部分。
- 在继续下一步之前彻底测试每个组件。
- 牢记安全性——验证输入并适当限制访问。
- 为代码编写良好的文档以便未来的维护。
- 仔细遵循MCP协议规范。

## 下一步

在Claude帮助您构建服务器后：

1. 仔细检查生成的代码。
2. 使用MCP Inspector工具测试服务器。
3. 将其连接到Claude.app或其他MCP客户端。
4. 根据实际使用和反馈进行迭代。

请记住，当需求随时间变化时，Claude可以帮助您修改和完善服务器。

需要更多指导吗？只需向Claude提出关于实现MCP功能或解决出现的问题的具体问题即可。
